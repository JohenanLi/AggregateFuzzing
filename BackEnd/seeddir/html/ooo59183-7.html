Document Owner:
Christian Lohmaier

Last Document Change:
02. Aug. 2006

Status of Document:
Preliminary

Test purpose

Verify that the Slackware-menus package can be installed and uninstalled and that Desktop-Integration works properly.

Known issues:

  • none

Preconditions of the test:

  • Have Slackware installed
  • Have the main OpenOffice.org packages (application) installed

Test documents

  • none

Test cases

Installation
  • Become root (run su)
  • run installpkg openoffice.org-slackware-menus-<mainversion>-noarch-<release>.tgz
To verify
  • Log in to KDE as regular user
  • Verify that you can launch OpenOffice.org Applications from the main KDE menu
  • Verify that you can open OpenOffice.org documents by simply clicking on them in the filemanager
  • Verify that you can launch OpenOffice.org from a terminal by running "soffice"
Deinstallation
  • Become root (run su)
  • run removepkg openoffice.org-slackware-menus
To verify
  • Log in to KDE as regular user
  • There should no longer be launchers for OpenOffice.org Applications in the main KDE -menu
  • the link /etc/openoffice.org-<version> should have been removed
  • You should no longer be able to launch OpenOffice.org from a terminal by running "soffice" (specifying the full path - e.g. /opt/openoffice.org2.0/program/soffice - should still work)

Problematic areas

If you did install while being logged-in to KDE, the changes might not be in effect immeadiately - you might have to logout of KDE and then log in to KDE again.
